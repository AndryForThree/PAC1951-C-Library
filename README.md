# PAC1951 C Driver

C driver for the Microchip PAC1951 power monitor.

This library provides a small, readable driver for the PAC1951, with support for voltage, current and power measurements, energy and charge accumulation, measurement range configuration, sample mode configuration, accumulator mode configuration, and alert setup.

The driver was originally written for a TI microcontroller project, but the PAC1951 logic is kept mostly independent from the MCU-specific code. Porting it to another platform mainly requires replacing the low-level I2C and delay functions.

## Current Scope

This version is focused on the PAC1951 single-channel device.

The code is written with the PAC195x family in mind, so some parts are intentionally structured in a way that should make future PAC1952/PAC1953/PAC1954 support easier. However, multi-channel support is not implemented yet.

## Refresh Behavior

The PAC1951 has different refresh commands, and this matters especially when using the accumulator.

This driver uses:

- `PAC1951_refreshV()` when the goal is to update readable measurement/control values without clearing the accumulator.
- `PAC1951_refresh()` when the accumulator must be reset, for example after changing range, accumulator mode, or after reading accumulated energy/charge.

This distinction is important because a full refresh clears the accumulation history.

## Accumulation Notes

Energy and charge readings rely on the PAC1951 accumulator.

The driver tracks whether the current accumulation period is valid through an internal `accumulationValid` flag. This avoids returning energy or charge values when the PAC1951 has been in a sample mode that makes the simple accumulation conversion invalid.

At the moment, `PAC1951_readEnergy_uWh()` and `PAC1951_readCharge_uAh()` are intended for adaptive accumulation modes, where the effective accumulation rate is treated as `1024 samples per second`.

If the sample mode, range, accumulator mode, or active channel configuration changes, the driver resets or invalidates the accumulator as needed.

## Measurement Ranges

The driver supports the PAC1951 range modes:

```c
PAC1951_RANGE_UNIPOLAR_FSR
PAC1951_RANGE_BIPOLAR_FSR
PAC1951_RANGE_BIPOLAR_HALF_FSR
```

The range configuration affects the conversion from raw register values to physical units.

Use:

```c
PAC1951_setMeasurementRange(dev, vsenseRange, vbusRange);
```

to configure the device and keep the internal driver object synchronized.

## Alerts

The driver exposes alert setup in three separate steps:

1. Configure the pin mode.
2. Enable or disable alert sources.
3. Route alert sources to `ALERT1` or `ALERT2`.

This keeps the API readable while still allowing detailed control.

Example alert concepts supported by the enum:

- channel 1 over-current;
- channel 1 under-current;
- channel 1 over-voltage;
- channel 1 under-voltage;
- channel 1 over-power;
- accumulator overflow;
- accumulator count limit;
- conversion complete.

## Dependencies

This first version targets a TI MCU project and uses:

- `ti/drivers/I2C.h`
- `ti/sysbios/knl/Clock.h`
- `ti/sysbios/knl/Task.h`

The TI-specific parts are mostly limited to:

- `PAC1951_readRegister()`
- `PAC1951_writeRegister()`
- `PAC1951_sendCommand()`
- `PAC1951_delay_ms()`

To port the driver to another MCU, these are the main functions that should be adapted.


## Basic Usage Flow

A typical application should:

1. Open the I2C peripheral using the MCU vendor driver.
2. Initialize a `PAC1951_Object` with `PAC1951_init()`.
3. Verify the connected device with `PAC1951_verifyDevice()`.
4. Configure measurement range, sample mode and accumulator mode if needed.
5. Read measurements using the public read functions.

The PAC1951 I2C bitrate is configured by the MCU I2C peripheral, not by the PAC1951 driver. For normal I2C operation, the PAC1951 follows the clock generated by the I2C master.

## Advanced API

The advanced API gives direct control over the PAC1951 registers and refresh commands.

```c
PAC1951_Status PAC1951_sendCommand(PAC1951_Object *dev, uint8_t commandReg);

PAC1951_Status PAC1951_refresh(PAC1951_Object *dev);
PAC1951_Status PAC1951_refreshV(PAC1951_Object *dev);

PAC1951_Status PAC1951_readRegister(PAC1951_Object *dev,
                                    uint8_t reg,
                                    uint8_t *data,
                                    uint8_t len);

PAC1951_Status PAC1951_writeRegister(PAC1951_Object *dev,
                                     uint8_t reg,
                                     const uint8_t *data,
                                     uint8_t len);
```

Use these functions when:

- a register must be written in one operation;
- timing matters;
- a feature is not wrapped yet by the normal public API;
- debugging the PAC1951 configuration;
- porting or extending the driver.

For normal measurement and configuration tasks, the standard public API is recommended.

## Files

- `PAC1951.h`

- `PAC1951.c`
